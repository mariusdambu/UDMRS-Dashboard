Filosofía RC madura

La estructura oficial única es Año / Trimestre. El perfil QuarterlyFolders es el único perfil válido. El modelo estructural anterior está retirado.

ComponenteRol actual
OrganizeConstruye/reconstruye biblioteca, clasifica, deduplica, repara EXIF si se pide y registra el índice.
NormalizeExistingFoldersReestructuración visual y limpieza: año/trimestre, carpetas vacías, junk-only y ramas residuales.
ReconcileProcessedDatabaseHerramienta principal para sincronizar/reparar ProcessedFiles.json.
PurgeMissingFromProcessedDatabaseAvanzado. Purga historial desaparecido después de revalidar.
DedupeCleanupDuplicados por hash. No limpia carpetas vacías.
-KeepEmptyFolders no limpia carpetas. Hace lo contrario: conserva carpetas vacías. Para permitir limpieza durante Normalize usa -KeepEmptyFolders:$false o no pases el parámetro.

Acceso desde dashboard

El flujo normal vive en Inicio: elegir origen/destino, revisar resumen dinámico y pulsar Ejecutar.

La entrada Importar galería vive como sección principal del dashboard. Úsala para exportaciones de proveedores con sidecars, JSON, álbumes, papelera o relaciones entre elementos. Para carpetas normales de fotos/vídeos usa Inicio.

En Ajustes, el botón Modo avanzado abre opciones avanzadas que antes requerían comandos PowerShell largos. No duplica acciones ya visibles como TestScan, Reconcile o Purge.

Botón avanzadoSwitch del motorResumen
Limpieza de retención-RetentionCleanupLimpia solo backups EXIF antiguos y duplicados confirmados antiguos. No limpia la galería normal ni carpetas vacías.
Recuperar duplicados mal movidos-RecoverFromWrongDuplicateMoveRecuperación conservadora desde duplicados usando logs e índice.
Traducir carpetas existentes-RenameExistingFoldersToCurrentLanguageTraduce carpetas trimestrales existentes al idioma activo sin leer contenido.
Traducir carpetas internas-RenameInternalFoldersToCurrentLanguageRenombra o consolida contenedores internos conocidos al idioma activo sin leer fotos ni hidratar nube.
Normalizar estructura-NormalizeExistingFoldersReestructura visualmente a Año/Trimestre y limpia ramas vacías/junk/residuales.
Limpieza de duplicados-DedupeCleanupDuplicados exactos por SHA256; no limpia carpetas vacías.
Reparar EXIF in-place-RepairOnlyExistingOrganizedLibrary -RepairExifReparación EXIF dentro de la biblioteca organizada; no mueve ni deduplica.
Migrar UDMRS a otro PCDashboard helperCrea ZIP de instalación compartida, ZIP de estado de usuario y MigrationGuide.txt. No incluye logs ni runtime.
Los botones avanzados respetan Simulación por defecto. Si activas Apply real, el dashboard pide confirmación específica y muestra los switches que va a lanzar.
Las acciones de Modo avanzado se ejecutan en una consola técnica persistente por usuario. El dashboard abre una sola ventana pwsh / powershell.exe, la reutiliza para siguientes acciones y no la cierra al terminar cada acción, para conservar salida, errores y progreso visible. Al cerrar el dashboard, esa consola técnica controlada y su estado temporal se cierran también. La infraestructura runtime vive en %APPDATA%\PhotoOrganizer\Runtime\TechnicalConsole; los logs y progress de las acciones se escriben en %APPDATA%\PhotoOrganizer\Logs.
Después de renames estructurales o Normalize Apply, Explorer y OneDrive pueden tardar unos minutos en refrescar, reindexar o sincronizar. Si el log confirma que no hubo content scan, EXIF, hash ni cloud traversal, ese estado temporal de Explorer no implica corrupción, hidratación masiva ni pérdida de datos. Espera antes de lanzar otra operación; si Explorer sigue bloqueado demasiado tiempo, reinicia Explorer o Windows.
Migración a otro PC. El botón avanzado Migrar UDMRS a otro PC genera paquetes en %USERPROFILE%\Downloads\UDMRS-MigrationPackages y abre esa carpeta al terminar: un ZIP de la instalación actual, un ZIP del estado del usuario actual y una guía. El paquete de usuario incluye ProcessedFiles.json, Config, IndexBackups y ajustes; excluye Logs, Runtime, *.progress.json, colas, PID/status temporales y backups EXIF locales.

Carpetas excluidas / protegidas externas

La aplicación separa carpetas internas propias de carpetas externas que el usuario decide proteger.

TipoSignificado
InternalProtectedFoldersCarpetas propias de la aplicación: logs, runtime, backups internos, revisión, duplicados y contenedores equivalentes.
UserExcludedFoldersCarpetas que el usuario declara intocables desde el dashboard.
VendorManagedFoldersPresets/ayudas para carpetas gestionadas por apps externas, por ejemplo Samsung Gallery o Camera Roll.

El dashboard muestra en Inicio un resumen de carpetas protegidas activas, encontradas y no encontradas, con acceso directo para gestionarlas sin editar JSON. La fuente activa unica es la configuración de cada usuario en %APPDATA%\PhotoOrganizer\Config\UserExcludedFolders.json. Si no existe, UDMRS la crea automáticamente sin exclusiones activas. Las rutas pueden usar %USERPROFILE%, <SourcePath>, <DestinationPath> y rutas absolutas.

Una carpeta excluida es una zona protegida: no se organiza, no se normaliza, no se repara EXIF, no se deduplica, no se purga, no se limpia, no se calcula hash, no se lee EXIF y no se hidrata nube. Si la ruta no existe, la entrada se conserva y se muestra como no encontrada.

Samsung Gallery se ofrece como preset sugerido, pero no se activa globalmente para todos los usuarios y no es un hardcode especial.

Uso normal

1. TestScan

Seguro. Escanea archivos compatibles, exclusiones y acceso. No lee EXIF, no calcula hashes, no mueve, no copia y no modifica datos.

  -SourcePath "%USERPROFILE%\OneDrive\Imágenes" `
  -DestinationPath "%USERPROFILE%\OneDrive\Imágenes\Fotos_Organizadas" `
  -TestScan `
  -Language es

18. RetentionCleanup Apply

Avanzado. Borra solo contenido antiguo apto para retención; conserva la última ejecución válida y no toca duplicados pendientes de revisión. Si no han pasado los días de retención configurados, puede no borrar nada; eso es normal.

  -Language es `

RetentionCleanup no es Normalize, no es Purge y no es limpieza general. Solo puede actuar sobre %LOCALAPPDATA%\PhotoOrganizer\_Backup_Metadate / aliases de metadata backup y sobre _Carantina_Duplicate_Confirmate / aliases de cuarentena confirmada. La carpeta de duplicados pendientes de revisión nunca se toca.

Logs, ETA y progress

Al arrancar, el dashboard limpia automáticamente artefactos operativos antiguos en %APPDATA%\PhotoOrganizer\Logs con una retención aproximada de 7 días: *.log, *.progress.json y reportes HTML. No toca fotos, vídeos, ProcessedFiles.json real, backups EXIF, cuarentenas ni %APPDATA%\PhotoOrganizer\Runtime\TechnicalConsole. Los backups técnicos del índice viven separados en %APPDATA%\PhotoOrganizer\IndexBackups; se conserva siempre el más reciente, se conservan backups recientes hasta un máximo aproximado de 10 copias y se migran automáticamente backups antiguos desde Logs\JsonBackups.

Ver último log

Get-ChildItem -LiteralPath "%APPDATA%\PhotoOrganizer\Logs" -Filter "PhotoOrganizer-*.log" |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 1 |
  Get-Content -Tail 80

Ver último progress.json

Get-ChildItem -LiteralPath "%APPDATA%\PhotoOrganizer\Logs" -Filter "*.progress.json" |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 1 |
  Get-Content -Raw
